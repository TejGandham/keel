Design Principles

  • Cards size to content — clean repos are compact, problem repos are bigger
  • Left border = status color — green/blue/yellow/orange/red at a glance
  • Everything visible — dirty files, branch names, pull-blocked reasons, all inline
  • Action buttons only where relevant — clean repos don't need buttons cluttering them
  • No click-to-reveal — cockpit view, zero hidden state
